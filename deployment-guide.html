🚀 Deployment Guide

Black Stars CRM · Going from offline-only to cloud-synced

📌 Two phases:
  1. Set up Firebase (data storage + login) — about 15 minutes
  2. Deploy the app to a free static host so anyone can open it via URL — about 5 minutes

Phase 1 — Set up Firebase

Step 1: Create the Firebase project

  1. Go to console.firebase.google.com and sign in with a Google account (any Gmail account works).
  2. Click + Add project.
  3. Name it blackstars-crm (or anything you want). Click Continue.
  4. When asked about Google Analytics — turn it OFF (you don't need it). Click Create project.
  5. Wait ~30 seconds for the project to be created. Click Continue.

Step 2: Register the Web App

  1. On the project's home page you'll see icons for iOS, Android, and Web. Click the </> (Web) icon.
  2. Nickname: Black Stars Web. Don't check "Also set up Firebase Hosting" yet. Click Register app.
  3. Firebase shows you a code block with a firebaseConfig object that looks like this: 
    const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "blackstars-crm.firebaseapp.com",
  projectId: "blackstars-crm",
  storageBucket: "blackstars-crm.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abc..."
};
  4. Copy those 6 values. Open firebase-config.js in the app folder and paste each value into the matching field. Save the file.
  5. Back in the Firebase Console, click Continue to console.

Step 3: Enable Firestore Database

  1. In the left sidebar: Build → Firestore Database.
  2. Click Create database.
  3. Pick a location: asia-south1 (Mumbai) — closest to Qatar, fast response time. This cannot be changed later! Click Next.
  4. Pick Start in production mode. Click Create.
  5. Once created, click the Rules tab. Replace the rules with this: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Only authenticated users can read or write
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
  6. Click Publish.

Step 4: Enable Authentication

  1. In the left sidebar: Build → Authentication.
  2. Click Get started.
  3. Pick Email/Password from the list. Toggle "Enable" ON. Don't enable email-link sign-in. Click Save.
  4. Click the Users tab → Add user.
  5. Email: admin@blackstars.qa (or whatever you want). Password: pick a strong one and write it down. Click Add user.
✓ Firebase setup complete. Open index.html in a browser now — the login screen should say "☁️ Cloud sync enabled". Log in with the email/password you just created.

Phase 2 — Deploy the app online

You have two free options. Firebase Hosting is the simplest (same account, same project). Netlify Drop is fastest if you don't want command-line tools.

Option A: Firebase Hosting (recommended)

  1. Install Node.js if you don't have it: nodejs.org (LTS version).
  2. Open a terminal in the app folder. Run: 
    npm install -g firebase-tools
firebase login           # opens browser to sign in
firebase init hosting    # follow prompts:
  # - Use existing project → pick your blackstars-crm project
  # - Public directory: .  (just a dot, current folder)
  # - Single-page app: No
  # - Set up automatic builds with GitHub: No
  3. Deploy: 
    firebase deploy --only hosting
  4. Firebase prints a URL like https://blackstars-crm.web.app. Open it — your app is live.
  5. Share this URL with the admin. He can bookmark it and use it from any device.

Option B: Netlify Drop (no terminal needed)

  1. Go to app.netlify.com/drop.
  2. Drag the entire app folder into the page. Wait 20 seconds.
  3. Netlify gives you a URL like https://random-name-123456.netlify.app.
  4. Click "Site settings" to rename it to blackstars-crm.netlify.app if you want.
⚠️ Important — authorize your domain in Firebase Auth. After deploying, go to Firebase Console → Authentication → Settings tab → Authorized domains. Add your deployment URL (e.g. blackstars-crm.web.app or blackstars-crm.netlify.app). Otherwise login will fail with "domain not authorized".

What happens after deployment

  • Admin opens https://blackstars-crm.web.app in any browser (laptop, phone, tablet).
  • Logs in with the email/password you created.
  • All data is in Firestore — same data everywhere.
  • If admin's phone has no internet, the app keeps working offline thanks to Firestore's offline persistence. When internet comes back, changes sync automatically.
  • If you (the owner) log in from your laptop while admin is working from his phone, both see real-time updates.

Updating the app later

When I send you a new version zip:

  1. Unzip on your computer.
  2. Copy the firebase-config.js you customized from the OLD version folder into the NEW version folder. (Otherwise cloud sync will turn off because the new zip has an empty config.)
  3. Deploy: firebase deploy --only hosting (Option A) or drag-drop on Netlify Drop (Option B).
  4. Admin reloads the browser — new version is live. His data is untouched because it lives in Firestore, not in the app code.
✓ Pro tip: Keep your firebase-config.js safe. It's the only file you need to preserve between updates. Everything else in the app folder is replaceable.

Troubleshooting

Login says "domain not authorized"

Firebase Console → Authentication → Settings tab → Authorized domains → Add your deployment URL.

App shows "💾 Offline mode" instead of "☁️ Cloud sync"

Check firebase-config.js — the apiKey field must not be empty. Open browser DevTools (F12) → Console tab — Firebase init errors will appear there.

Changes don't appear on the other device

Make sure both devices are logged in with the SAME email/password. If one is on a stale browser cache, hard-reload (Ctrl+Shift+R).

I want to migrate my existing localStorage data to the cloud

  1. On the device with your existing data: Settings → 💾 Backup all data (JSON). Save the file.
  2. Edit firebase-config.js and reload — now you're in cloud mode. The app shows empty (Welcome panel).
  3. Log in with your Firebase user.
  4. Settings → 📂 Restore from backup → pick the JSON. Done.

I want to switch back to offline mode

Delete the contents of all apiKey/authDomain/etc. fields in firebase-config.js. The app falls back to localStorage on next reload.
